Exécuter le ML avec TensorFlow

Aperçu

Dans cet atelier, vous allez utiliser le bloc-notes Datalab de l'atelier précédent pour créer un réseau de neurones TensorFlow et prévoir la demande de taxis.

Objectifs de l'atelier

Au cours de cet atelier, vous allez apprendre à réaliser l'opération suivante :

Créer un réseau de neurones et prévoir le niveau de la demande de taxis à New York à l'aide de TensorFlow

Présentation

Dans cet atelier, vous allez concevoir un système de prévision du niveau de la demande de taxis en utilisant le machine learning. Dans l'atelier précédent, vous avez créé un ensemble de données d'entraînement répertoriant les différents facteurs qui influent sur le niveau de demande de taxis (les "éléments d'entrée") ainsi que le niveau de demande réel (la "cible") des jours précédents. Dans cet atelier, vous allez utiliser ces données pour entraîner un réseau de neurones TensorFlow capable de prévoir le niveau de demande des jours à venir à l'aide des éléments d'entrée dont il dispose.

Configuration

Tâche 1 : Lancer Cloud Datalab

Pour lancer Cloud Datalab, procédez comme suit :

  1. Dans Cloud Shell, saisissez la commande suivante :
datalab create bdmlvm --zone us-central1-a

Le lancement de Datalab prend environ cinq minutes.

Remarque: Suivez les instructions à l'écran pour cette étape.

Tâche 2 : Consulter le bloc-notes de Cloud Datalab

Si nécessaire, patentiez jusqu'à la fin du lancement de Datalab. Vous saurez que l'outil est prêt à l'emploi lorsque l'instruction "Web Preview" (Aperçu sur le Web) s'affichera.

  1. Cliquez sur l'icône Aperçu sur le Web dans l'angle supérieur droit du ruban Cloud Shell. Cliquez sur Modifier le port. Choisissez le port 8081 dans la boîte de dialogue Modifier le port d'aperçu , puis cliquez sur Modifier et prévisualiser.

    Remarque: Vous restez connecté à votre instance Datalab tant que la commande datalab est active. Si l'instance Cloud Shell utilisée pour l'exécution de la commande est fermée ou interrompue, vous êtes déconnecté de votre VM Cloud Datalab. Le cas échéant, essayez de vous reconnecter à l'aide de la commande datalab connect bdmlvm dans une nouvelle instance Cloud Shell.

  2. Dans l'angle supérieur droit du ruban Datalab, cliquez sur l'icône Open ungit (Ouvrir ungit).

  3. Dans la fenêtre Ungit, supprimez notebooks dans le nom /content/datalab/notebooks de sorte à obtenir le nom de chemin suivant : /content/datalab. Appuyez ensuite sur Enter.

  4. Dans le panneau qui s'ouvre, pour indiquer à partir de quel dépôt GitHub effectuer le clonage, saisissez le texte suivant dans le champ Clone from (Cloner à partir de) :

    https://github.com/GoogleCloudPlatform/training-data-analyst

  5. Cliquez sur Clone repository (Cloner le dépôt).

Tâche 3 : Ouvrir un bloc-notes Datalab

  1. Dans le navigateur Datalab, accédez à training-data-analyst > CPB100 > lab4a > demandforecast.ipynb.

  2. Comme pendant l'atelier précédent, exécutez les cellules du bloc-notes jusqu'à ce que vous arriviez à la section Machine Learning with TensorFlow (Machine learning avec TensorFlow).

  3. Continuez à exécuter le code dans les cellules restantes.

  4. Lisez le commentaire, cliquez sur Clear | Clear all Cells (Effacer > Effacer toutes les cellules), puis exécutez les extraits de Python dans la cellule un par un. Exécutez chaque segment de code avec le raccourci clavier Shift+Enter.
