Cree un conjunto de datos de AA con BigQuery

Descripción general

En este lab, usará BigQuery desde Datalab para crear un DataFrame de Pandas que se usará como datos de entrenamiento para la predicción de la demanda de taxis en una ciudad.

Qué aprenderá

En este lab, aprenderá a realizar las siguientes actividades:

  • Cómo utilizar BigQuery y Datalab para explorar y visualizar datos

  • Cómo crear un DataFrame de Pandas que se usará como el conjunto de datos de entrenamiento para el aprendizaje automático en TensorFlow

Introducción

En este lab y el siguiente, creará un sistema de previsión de la demanda mediante aprendizaje automático. El propósito es utilizar TensorFlow para predecir la demanda diaria de taxis en una ciudad. Para ello, necesitará un conjunto de datos de entrenamiento que incluya los factores que afectan la demanda de taxis (“atributos de entrada”) y la demanda real (el “objetivo”) durante un período anterior compuesto de varios días. Usará estos datos para entrenar una red neuronal capaz de predecir la demanda diaria en el futuro a partir de esos atributos de entrada.

En este lab, usará BigQuery en Datalab para crear el conjunto de datos que se empleará en el siguiente lab a fin de llevar a cabo el aprendizaje automático.

Configuración

Tarea 1: Inicie Cloud Datalab

Siga estos pasos para iniciar Cloud Datalab:

  1. En Cloud Shell, escriba lo siguiente:

    datalab create bdmlvm --zone us-central1-a

Datalab tardará unos 5 minutos en iniciarse.

Nota: Recuerde seguir las indicaciones durante el proceso.

Si no conoce Datalab, a continuación encontrará un gráfico con las funciones principales del programa:

Tarea 2: Registre el notebook en Cloud Datalab

Nota: Si es necesario, espere hasta que Datalab termine de iniciarse. Datalab estará listo cuando vea un mensaje que le sugiera usar una “Web Preview”.

  1. Haga clic en el ícono de vista previa Web, ubicado en la esquina superior derecha de la cinta de Cloud Shell. Haga clic en Cambiar puerto. Cambie al puerto 8081 con el cuadro de diálogo Cambiar puerto de vista previa y, luego, haga clic en cambiar obtener vista previa.

    Nota: La conexión a su instancia de Datalab se mantiene abierta mientras permanezca activo el comando datalab. Si se cierra o interrumpe la instancia de Cloud Shell utilizada para ejecutar el comando datalab, perderá la conexión con su VM de Cloud Datalab. Si esto ocurre, tal vez pueda restablecer la conexión mediante el comando datalab connect bdmlvm en una nueva instancia de Cloud Shell.

  2. En Datalab, haga clic en el ícono Open ungit ubicado en la esquina superior derecha.

  3. En la ventana de Ungit, seleccione el texto que dice /content/datalab/notebooks, quite la palabra notebooks para que quede /content/datalab y presione la tecla Enter.

  4. En el panel que aparecerá, escriba lo siguiente para indicar el repositorio de GitHub que Clone from:

    https://github.com/GoogleCloudPlatform/training-data-analyst

  5. Haga clic en Clone repository.

Tarea 3: Abra un notebook de Datalab

  1. En el navegador de Datalab, vaya a training-data-analyst > CPB100 > lab4a > demandforecast.ipynb.

  2. Lea los comentarios y haga clic en Clear | Clear all cells. Luego, ejecute los fragmentos de Python en la celda, paso a paso (use Shift+Enter para ejecutar cada parte del código).

  3. Deténgase cuando llegue a la sección Machine Learning with Tensorflow, ya que ese es el tema del próximo lab.
